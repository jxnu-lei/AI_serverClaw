# AI Terminal

智能服务器管理终端，提供统一的服务器连接、会话管理和AI辅助功能。

## 项目简介

AI Terminal 是一个现代化的服务器管理工具，集成了终端连接、会话管理和AI助手功能，旨在简化服务器管理流程，提高运维效率。

## 功能特性

### 核心功能
- **服务器连接管理**：支持SSH、Telnet等多种协议，统一管理服务器连接
- **终端会话**：提供Web-based终端，支持多标签页管理多个会话
- **会话录制**：自动录制终端会话，支持回放和审计
- **AI助手**：集成AI聊天功能，可帮助解决技术问题
- **用户管理**：支持多用户，基于角色的权限控制
- **系统设置**：个性化配置系统参数

### 技术特性
- **前后端分离**：Vue 3 + TypeScript 前端，FastAPI 后端
- **响应式设计**：适配不同屏幕尺寸
- **实时通信**：WebSocket 实现实时终端通信
- **安全认证**：JWT 令牌认证，密码加密存储
- **数据库**：SQLite 轻量级数据库

## 安装部署

### 环境要求
- Python 3.10+
- Node.js 16+
- npm 或 yarn 或 pnpm

### 后端部署

1. **进入后端目录**
   ```bash
   cd server
   ```

2. **创建虚拟环境**
   ```bash
   python -m venv .venv
   ```

3. **激活虚拟环境**
   - Windows:
     ```bash
     .venv\Scripts\activate
     ```
   - Linux/Mac:
     ```bash
     source .venv/bin/activate
     ```

4. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

5. **启动后端服务器**
   ```bash
   python -m uvicorn app.main:app --reload
   ```
   后端服务将运行在 `http://127.0.0.1:8000`

### 前端部署

1. **进入前端目录**
   ```bash
   cd client
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```
   前端服务将运行在 `http://localhost:5173`

4. **构建生产版本**
   ```bash
   npm run build
   ```
   构建产物将生成在 `dist` 目录

## 使用指南

### 登录
1. 访问 `http://localhost:5173/login`
2. 使用默认管理员账号登录：
   - **用户名**：admin
   - **密码**：admin!123

### 服务器连接
1. 在左侧菜单中选择 **工作台**
2. 点击 **新建连接** 按钮
3. 填写服务器连接信息：
   - 连接名称
   - 主机地址
   - 端口
   - 用户名
   - 认证方式（密码或密钥）
4. 点击 **连接** 按钮建立连接

### AI 助手
1. 在左侧菜单中选择 **系统设置** > **AI 助手**
2. 配置 AI 提供商和 API Key
3. 保存配置后，返回 **AI 助手** 页面
4. 在聊天框中输入问题，点击发送按钮
5. AI 将返回回答和建议

### 用户管理
1. 在左侧菜单中选择 **系统设置** > **用户管理**
2. 点击 **新建用户** 按钮
3. 填写用户信息：
   - 用户名
   - 邮箱
   - 密码
   - 角色（admin 或 user）
4. 点击 **保存** 按钮创建用户

## 安全注意事项

1. **密码安全**：使用强密码，并定期更换
2. **API Key**：妥善保管 AI 提供商的 API Key，避免泄露
3. **服务器凭证**：避免在公共网络环境中输入服务器密码
4. **会话管理**：长时间不使用时，及时退出登录
5. **权限控制**：根据用户职责分配适当的角色权限

## 常见问题

### 登录失败
- 检查用户名和密码是否正确
- 检查后端服务器是否运行
- 检查网络连接是否正常

### 终端连接失败
- 检查服务器地址和端口是否正确
- 检查用户名和密码/密钥是否正确
- 检查服务器是否允许远程连接
- 检查防火墙设置

### AI 助手无响应
- 检查 AI 提供商配置是否正确
- 检查 API Key 是否有效
- 检查网络连接是否正常
- 检查 AI 提供商服务是否正常
